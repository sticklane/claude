# Evidence 05: end-to-end dispatch-language check

## Criterion

Captured `/parallel` dispatch prompts containing **tier** and **output-budget**
language, observed in the dispatch text *before any worker ran*.

## Method

The `/parallel` skill (`.claude/skills/parallel/SKILL.md`) is
`disable-model-invocation: true` and its dispatch tooling (background
`general-purpose` workers in isolated worktrees) is not available to an
unattended worker, so no live workers were launched. Per the task's binding
note, the criterion is satisfied by capturing the *authored* dispatch-prompt
prose that `/parallel` emits before any worker runs — the retrofit (wte-03)
baked the tier + output-budget language directly into that worker prompt. The
capture below is the verbatim dispatch text from the skill's step 2
("## 2. Dispatch"), which is the exact string substituted per task and handed
to each background agent at launch.

## Captured dispatch prompt (verbatim, parallel/SKILL.md step 2)

> Execute the task in <task-file> following the build skill's procedure
> exactly, as written in <build-skill-path> (resolved at dispatch):
> delegate mechanical scouting to Haiku (`effort: low`) scouts for exploration, tests first
> where criteria are test-shaped, run every acceptance command, standard
> gates, then commit to a branch named task/NN-<slug>. The task file's
> `Budget:` line is a ceiling, not a target: when remaining work clearly
> exceeds the remaining budget, stop with verdict BLOCKED "over budget"
> rather than grind on. Your final message
> must be: verdict (DONE/BLOCKED), acceptance evidence per criterion (command
> + result), branch name, and files changed. If blocked, say why and stop —
> do not improvise around the task's scope.

## Assertion

- **Tier language present** — "delegate mechanical scouting to Haiku
  (`effort: low`) scouts for exploration" names the mechanical-stage model tier
  (Haiku / low effort) for scouting, per the "Tier by stage type" rule in
  `.claude/rules/token-discipline.md`.
- **Output-budget language present** — "Your final message must be: verdict
  (DONE/BLOCKED), acceptance evidence per criterion (command + result), branch
  name, and files changed" caps each worker's return to a structured verdict
  rather than a transcript, per the "Cap subagent returns" rule.
- Both appear in the authored dispatch prose, i.e. in the text every worker
  receives at launch — before any worker executes.

The ultra path (step "## Ultra path") reuses this same worker prompt: "one
worker per task file (worktree isolation, the same worker prompt plus
effort-tier language)", so the tier + budget language holds on that path too.
