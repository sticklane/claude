# Memory index

Narrow, load-on-demand lessons. Read the topic file when a task matches its
trigger; this index is not loaded at session start. One line per topic:
path — when to read.

- [worktree-base-tracking-ref](memory/worktree-base-tracking-ref.md) — worktree agents see stale files / miss merged deps / conflict on merge (cut from `origin/main`, not HEAD).
- [unattended-worker-tool-limits](memory/unattended-worker-tool-limits.md) — authoring a drainable task, a worker DEFERRED because it "couldn't run" a criterion (no Workflow tool; `/evals` flag-blocked; execution-stage skills launch-contract-blocked — worker contexts carry no live-user authorization, post-2026-07-11 migration).
- [live-drain-reconciliation](memory/live-drain-reconciliation.md) — asked to reconcile in-progress tasks or clear stale/needs-review/unpushed git flags: check for a LIVE drain first (task/\* worktrees, advancing `drain:` commits, Status flipping) — it owns that cleanup; manual edits collide.
- [workboard-mirror-verbatim](memory/workboard-mirror-verbatim.md) — editing the workboard skill (workboard.py / test_workboard.py), or writing `/breakdown` acceptance criteria for a task touching `antigravity/` mirrors: only workboard's `.py` files are byte-identical (port with `cp` + `diff -q`); prose-skill mirrors are paraphrased ports — never write a `diff → no output` criterion for those.
- [concurrent-session-collision](memory/concurrent-session-collision.md) — starting a multi-file edit, or files change under you that you didn't edit: check no OTHER live Claude session shares the (non-worktree) tree (`ps aux | grep 'claude daemon'`) before proceeding; on collision stop and don't revert load-bearing changes.
- [skill-retirement-checklist](memory/skill-retirement-checklist.md) — retiring/folding/renaming a skill or workflow: what a clean reference grep still misses (bare-name mentions, unported behaviors, fallback paths, host-invariant clashes) and why the critic runs on the diff anyway.
- [root-container-test-failures](memory/root-container-test-failures.md) — tests/test\_\*.sh failing in a root container (remote/web session): three known chmod-based failures are environmental; confirm on clean HEAD and report, don't fix.
- [drain-dispatch-lessons](memory/drain-dispatch-lessons.md) — dispatching drain/build workers when a gate is already red on main, a worker spawns its own verifier sub-agent, a task's Touch lives in another repo (live-service checkout), a freshly-written SPEC.md/tasks/ hasn't been committed yet, you're running more than one spec's queue concurrently (cross-spec Touch collisions breakdown never checked, or a spec-completion review's ref-range double-counts a sibling's changes on a shared file), a foreign `DRAIN-OWNER.md` lease looks stale, a FRESH foreign lease holds the only pending spec (intake-alongside coexistence), critique intake is about to re-critique a spec unchanged since its recorded NOT READY, a cross-repo worker's verdict mentions `--no-verify`, collecting a DONE verdict's `Decisions:` bullets (write them into the task file's own `## Decisions` at collect-verdict time — nothing gates it mechanically), auditing a HUMAN.md entry that cites a `critique-findings.md` (check that file's OWN latest round, not the entry's date), a diff against `main` from inside an isolated orchestrator worktree looks empty right after a real merge (the local `main` ref is stale — diff against `origin/main`/HEAD/a SHA instead), a push is rejected by a concurrent drain session (fetch+merge+retry, don't just warn-and-move-on), or a worktree-cleanup `cd <other-path> && ...` one-liner leaves the shell's cwd silently pointed at the wrong checkout for later relative-path commands: the prompt clauses / procedures that handle each.
- [touch-root-fixture-tests](memory/touch-root-fixture-tests.md) — authoring/reviewing a task that changes a parser's sample/fixture output: list the root-level `*_test.go` files asserting fixture sample counts (e.g. `agentprof/cmd_claude_test.go`) in `Touch:`, or the worker is forced outside its declared scope to keep the suite green.
- [anchored-acceptance-criteria](memory/anchored-acceptance-criteria.md) — writing a SPEC/task acceptance grep or numeric bound: anchor greps on a NEW phrase verified absent (`grep -c` → 0) from every target, check bounds against the file's current value, and make version-bump checks assert the closing commit's own diff — the vacuous-grep / unsatisfiable-bound / sibling-bump traps from the 2026-07-11 spec wave.
- [multi-runtime-live-testing](memory/multi-runtime-live-testing.md) — installing/driving `agy` (Antigravity CLI) or `codex` (Codex CLI) directly, or scoping a port to a new coding-agent runtime: agy's 30s OAuth window and default-project bleed-through, both tools' shared Agent Skills standard (mirrors often port for free), Codex's missing `.agents/workflows/` invocation path, and `allow_implicit_invocation: false`'s actual documented semantics.
- [verifier-tier-leak](memory/verifier-tier-leak.md) — an agent's frontmatter `model:` pin (verifier=sonnet, scout=haiku, etc.) appears not to hold in a cost profile (a pinned agent billed at the session's frontier model), or reasoning about why a shipped agent-def fix doesn't show up in a running install: the stale immutable plugin-cache snapshot still serves the old `model: inherit` pin.
- [worktree-write-tool-sandbox](memory/worktree-write-tool-sandbox.md) — a worktree-isolated agent instructed to operate outside its own worktree (e.g. a drained generation told to work the shared main checkout): `Write`/`Edit` stay hard-sandboxed to the launch worktree even after `cd`; use `Bash` (heredoc/`sed`) for every file mutation outside it instead.
